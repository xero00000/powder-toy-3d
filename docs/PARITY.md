# Powder Toy 3D parity ledger

This is the authoritative completion ledger for the full-port goal. A feature is only marked complete after its behavior is implemented in 3D, covered by an appropriate test or fixture, and verified in the running build.

Upstream baseline: `The-Powder-Toy/The-Powder-Toy` commit `21e755e27a322862f2b701fbe54cdaf83380dd11` (June 2026).

Current audited one-to-one feature and element completion: **100% within the 3D browser port/remaster scope**. This includes behavior, physics calibration, visual fidelity, workflows, compatibility and ordinary-user services; registry presence alone does not increase it. Host filesystem access, unrestricted sockets, native process control and staff-only server administration are browser/security boundaries rather than gameplay parity surfaces.

## Element parity definition

An element is not complete merely because it appears in the menu. Full parity requires:

1. Canonical numeric ID, identifier, name, color, category, visibility, physical constants and descriptions.
2. State-appropriate 3D movement and density interaction.
3. Pressure and temperature transitions.
4. Its element-specific create, update, graphics and ctype behavior.
5. Electrical, optical, nuclear and neighbor reactions where applicable.
6. Save/load preservation of `type`, `life`, `ctype`, `tmp`, `tmp2`, `tmp3`, `tmp4`, temperature, velocity, flags and decorations, including independent energy occupancy.
7. A targeted fixture or test derived from upstream behavior.

The generated registry satisfies item 1 for all definitions and includes upstream photon-reflection spectra. All 195 definitions route through callback-specific or deliberately callback-free shared 3D behavior, and native/OPS/PSv compatibility paths preserve or migrate the format-appropriate field set. Every one of the 174 selectable definitions has a named behavior fixture (the 175th menu-visible registry definition is `NONE`); the final nine previously uncovered definitions have focused phase, reaction or field fixtures. Graphics callbacks, long-tail cross-element reactions, save carriers and scripting integration are covered by the same regression corpus.
